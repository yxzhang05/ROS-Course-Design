# odom_navi — 纯里程计 PID 坐标导航节点

## 简介

`odom_navi.py` 是一个**不依赖地图和激光雷达**的坐标导航节点。  
以节点启动时机器人所在位置为坐标原点 `(0, 0)`，用户在终端输入目标坐标 `(x, y)` 后，节点使用 **PID 算法**控制机器人移动到指定位置，最大线速度为 **0.1 m/s**。

适用场景：
- 不需要建图的短距离精确移动
- 调试里程计精度
- 无激光雷达的简化导航场景

---

## 代码逻辑说明

### 传感器使用

| 传感器 | ROS 话题 | 使用方式 |
|--------|---------|---------|
| 车轮里程计 | `/odom` (`nav_msgs/Odometry`) | 读取机器人相对出发点的位置 `(x, y)` |
| IMU 陀螺仪 | `/imu` (`sensor_msgs/Imu`) | 读取更精确的偏航角（优先于里程计偏航角，可关闭） |
| 速度控制 | `/cmd_vel` (`geometry_msgs/Twist`) | 发布线速度和角速度指令 |

> **注意**：`/odom` 由底盘驱动节点（STM32）发布，`/imu` 同样由底盘驱动发布。  
> 若 EKF（`robot_pose_ekf`）正在运行，里程计已融合了 IMU 数据，定位精度更高。

---

### 坐标系

```
原点 (0,0): 节点启动时机器人的实际位置
X 轴: 里程计坐标系 X 方向（通常为机器人启动时的前方向）
Y 轴: 里程计坐标系 Y 方向（通常为左方向）
```

例如，发送目标 `(1.0, 0.0)` 表示"沿里程计 X 轴正方向前进 1 米"。

---

### 状态机流程

```
        ┌──────────────────────────────────────────┐
        │           用户输入 (x, y)                 │
        └─────────────────┬────────────────────────┘
                          ▼
                    ┌──────────┐
                    │  IDLE    │  等待目标输入
                    └────┬─────┘
         收到目标         │
                          ▼
                    ┌──────────┐
                    │ROTATING  │  原地旋转，对准目标方向
                    │          │  角度 PID，线速度 = 0
                    └────┬─────┘
         角度误差 < 容差   │
                          ▼
                    ┌──────────┐
                    │ MOVING   │  直线前进到目标点
                    │          │  线速度 PID + 角度修正 PID
                    └────┬─────┘
         距离 < 容差      │           朝向偏差过大
                          │      ◄────────────────────┤（重新转向）
                          ▼
                    ┌──────────┐
                    │  DONE    │  到达目标点，停车
                    └────┬─────┘
                          │  等待下一个目标输入
                          ▼
                    ┌──────────┐
                    │  IDLE    │
                    └──────────┘
```

### PID 控制器设计

节点使用两个独立的 PID 控制器：

#### 角速度 PID（控制旋转）

```
误差 = 目标朝向角 − 当前偏航角   （单位：rad）
输出 = 角速度 angular.z          （单位：rad/s，限幅 ±1.0）

ROTATING 阶段：对准目标方向，linear.x = 0
MOVING 阶段：实时修正前进方向偏差
```

#### 线速度 PID（控制前进速度）

```
误差 = 到目标点的欧氏距离         （单位：m）
输出 = 线速度 linear.x            （单位：m/s，限幅 0 ~ 0.1）

距离越远速度越快，接近目标时自动减速
```

#### PID 公式

```
output = Kp × error
       + Ki × ∫error dt    （积分，带限幅防饱和）
       + Kd × Δerror/Δt   （微分）
```

---

### IMU 偏航角融合

当 `/imu` 话题可用时（`use_imu_yaw=true`，默认开启）：
- 记录节点启动时的 IMU 偏航角为参考值
- 用 `当前IMU偏航角 − 初始IMU偏航角` 作为相对偏航角
- 这比轮式里程计的偏航角更准确（陀螺仪漂移较小）

若 `/imu` 不可用，自动退回使用里程计偏航角。

## 使用说明

### 前提条件

1. ROS Noetic（或 Melodic）已安装
2. 仓库已编译：`catkin_make`（在工作空间根目录）
3. 机器人底盘通过 USB 串口连接，设备文件为 `/dev/car_driver_tty`

### 启动方式

**方式一：使用 launch 文件（推荐）**

```bash
# 启动底盘驱动 + 导航节点（新开 xterm 终端用于输入坐标）
roslaunch wulina odom_navi.launch
```

**方式二：分步启动**

```bash
# 终端 1：启动底盘驱动
roslaunch start_roscar start_roscar.launch

# 终端 2：启动导航节点
rosrun wulina odom_navi.py
```

**方式三：通过 rosrun 并传入参数**

```bash
rosrun wulina odom_navi.py _max_linear_vel:=0.08 _lin_kp:=0.6
```

---

### 操作方法

节点启动后，终端会显示：

```
当前位置: (0.000, 0.000)  输入目标坐标 "x y"，或输入 "q" 退出:
```

**输入格式**：`x y`（空格分隔，单位：米）

```
# 示例：向 X 方向前进 1 米
1.0 0.0

# 示例：向斜前方移动（X=0.5m, Y=0.5m）
0.5 0.5

# 示例：使用逗号分隔也可以
1.5,0.8

# 退出
q
```

机器人到达目标点后，终端会打印确认信息并再次提示输入下一个目标。

---

### 参数调整

在 launch 文件或 `rosrun` 命令行中可覆盖所有参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `~max_linear_vel` | `0.10` | 最大线速度 (m/s) |
| `~max_angular_vel` | `1.00` | 最大角速度 (rad/s) |
| `~dist_tolerance` | `0.05` | 到点判定距离容差 (m) |
| `~angle_tolerance` | `0.05` | 旋转对准角度容差 (rad) |
| `~use_imu_yaw` | `true` | 是否优先使用 IMU 偏航角 |
| `~heading_correct_threshold` | `0.35` | 前进中朝向偏差超过此值时停车重新对准 (rad) |
| `~lin_kp` | `0.50` | 线速度 PID 比例系数 |
| `~lin_ki` | `0.00` | 线速度 PID 积分系数 |
| `~lin_kd` | `0.08` | 线速度 PID 微分系数 |
| `~ang_kp` | `2.00` | 角速度 PID 比例系数 |
| `~ang_ki` | `0.00` | 角速度 PID 积分系数 |
| `~ang_kd` | `0.20` | 角速度 PID 微分系数 |

**示例：更改最大速度和 PID 参数**

```bash
roslaunch wulina odom_navi.launch max_linear_vel:=0.08 lin_kp:=0.6 ang_kp:=1.8
```

---

### PID 调参建议

| 现象 | 调整方向 |
|------|---------|
| 停车时超出目标点（过冲）| 减小 `lin_kp`，增大 `lin_kd` |
| 到达目标点太慢 | 增大 `lin_kp` |
| 旋转时来回摆动（振荡）| 减小 `ang_kp`，增大 `ang_kd` |
| 旋转后对准方向偏差大 | 增大 `ang_kp`，减小 `ang_kd` |
| 直线前进时偏向一侧 | 增大 `ang_kp`，减小 `heading_correct_threshold` |

## 局限性

1. **里程计漂移**：纯里程计定位会随时间/距离累积误差；实际可靠距离取决于路面条件、轮子打滑程度和里程计标定质量，在光滑硬地面上通常可达数米，在粗糙或打滑地面上会更快漂移。
2. **无障碍物检测**：本节点不检测障碍物，请确保路径畅通再发送目标。
3. **坐标系固定**：每次启动节点重新确定原点，关机后原点信息丢失。
4. **路面要求**：轮式里程计在光滑地面（打滑）或不平地面（颠簸）上精度较低。

## 与地图导航的对比

| 特性 | odom_navi（本节点）| navigation.launch（原有）|
|------|------------------|------------------------|
| 定位方式 | 纯里程计 + IMU | AMCL（激光雷达 + 地图）|
| 是否需要地图 | 否 | 是（需事先建图）|
| 是否需要激光雷达 | 否 | 是 |
| 障碍物检测 | 无 | 有（代价地图）|
| 长距离精度 | 较低（漂移累积）| 较高（激光雷达修正）|
| 启动复杂度 | 低（只启动底盘）| 高（底盘+雷达+地图+AMCL）|
| 适合场景 | 短距离、调试、简单任务 | 室内复杂环境长距离导航 |
